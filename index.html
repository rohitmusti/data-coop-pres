Clever Tricks

Scripting

We wrote a script that handles the landsat tile scraping and processing to calculate temperature

We wrote a GIS data pipeline (basically a massive jupyter notebook) that computes all of the calculations for us.

Speed Boosts

  • multi-threading the data downloading process
  • multi-processing the temperature tile analysis
  • incremental updates (only make updates if new tiles are added)

Hot Swap Tree Canopy

  • If we get a better data source, just by adding it to a folder, our main data pipeline will hot swap the old canopy for new canopy

Code & Data Stability

Myths

Myth: Thou Must Experience Big Tech

  • In big tech you build "sky-scrapers"
  • Tools like TES are more similar to houses
  • Building sky-scrapers do not make you better at building houses

Myth: Thou Must Be an "Expert"

  • Subject matter experts are very valuable!
  • However, not everyone on the team needs to be one
  • Don't let fear of not being a domain expert stop you

Myth: Build & Users Will Come

  • LOL
  • Get the word out!
  • Offer training sessions (automatic user research)!

Myth: Project Managers are Unnecessary

  • Project managers are excellent for maintaining project momentum
  • Running a good meeting is hard
  • Prep is work too

Myth: Thou Needs One More Week

  • You will always find something to fill that extra time with
    • refactoring, re-designing, new features, documentation

Myth: Thou Just Needs an Engineer

  • Most problems/decisions are not engineering problems
  • Without the in-house subject matter expertise, the great game of telephone begins

Obstacles

Data Quality and Documentation

  • Most data is not well documented or maintained
    • The TIGER/Line Census Block Group and Block shapefiles are slightly misaligned.
  • Maintaining thorough internal documents is important
  • Keep your raw data raw, all data transformations should be written in code
  • Extra QC is always required, spatial data requires visual checks on top of statistical checks.

Key Stone Team Member

If one person is solely responsible entire project parts,

  • documentation
  • transitions
  • updates

all become extremely difficult

Open Source Stability

  • A crucial trade off between building your own tools and functions vs using a library
  • Specifically, geopandas is still not perfect

Proprietary Software and Data Formats

  • ArcGIS is excellent in many ways, but its project scripting and data storage is not particularly portable.
  • @geodatabases & @arcpy
